# CaseBoard 隐私与数据说明

更新日期：2026-07-15

本说明描述 CaseBoard 在当前公开源码中的数据处理方式。不同功能是否发生外部传输，取决于用户启用的功能、选择的服务商和构建时配置。

## 1. 本地保存的数据

CaseBoard 会在本机应用数据目录及用户选择的位置写入数据，包括：

- SQLite 数据库中的案件、当事人、文档索引、结构化画像、期限、工作记录、聊天记录、测算记录和功能状态；
- 导入文件的路径及文件元数据。除用户明确执行导入、导出或生成操作外，应用不以“隐私承诺”为由保证所有功能均不复制文件；具体结果以操作界面提示为准；
- OCR/文本抽取结果、缓存、外部检索结果、报告、生成文书、诊断日志及更新状态；
- `settings.json` 等本地设置，其中可能包含用户填写的 API key、token、服务端点、OAuth 状态和稳定安装标识；
- 用户主动导出的 Word、HTML、Markdown、压缩包、知识库或案件资料包。

API key 和 token 不应提交到源码仓库，但当前实现可能将其保存在本机设置文件中。请通过操作系统账户、磁盘加密和目录权限保护本机数据。

## 2. 云端 LLM 与 OCR

当使用云端模型或云端 OCR 时，为完成用户发起的抽取、分析、聊天、写作或识别任务，应用可能向已配置的服务商发送：

- 文档全文、OCR 图片或原文件；
- 案件上下文、用户提示词、聊天内容及生成任务所需的结构化字段；
- 模型名称、请求参数、API key 和必要的技术元数据。

当前代码支持或兼容的服务包括 DeepSeek、MiniMax、OpenAI 兼容接口、MinerU、PaddleOCR 等。数据接收方的保留、训练、跨境和安全规则由对应服务条款及用户账户配置决定。处理敏感案件前，请先确认服务商规则及本单位授权。

## 3. 法律检索与第三方工具

- **元典**：法律、案例、企业和执行检索可能发送检索词、主体名称、证件号码、案号或其他用户选择的查询条件，并在本地缓存结果。
- **快递查询**：可能向快递服务发送运单号、快递公司及查询所需参数。
- **飞书与滴答清单**：启用后会按用户授权交换 OAuth 凭据、日历、日程或待办数据；同步内容以对应界面和第三方授权页为准。
- **MCP**：外部 MCP 客户端只有在应用内获得授权后才能访问相应能力；获准客户端可能读取或写入其权限范围内的案件数据。请只授权可信客户端，并及时撤销不再使用的授权。
- **其他自定义服务**：用户填写自定义端点或安装扩展能力时，数据会按该端点或能力的实现处理。

## 4. 更新、LPR 与网络请求

- **更新检查**：应用可能访问发布源以查询当前版本是否有更新。服务端通常能够看到常规网络元数据，例如 IP 地址、时间和 User-Agent；更新下载只在相应更新流程中发生。
- **人民银行 LPR 刷新**：自动或手动刷新会访问公开的人民银行来源，获取公布日期和利率数据，并在本地缓存；请求不需要发送案件内容。
- **打开外部链接**：用户点击官方来源或其他外部链接后，系统浏览器会直接访问对应网站。

## 5. 反馈

反馈功能不会自动把反馈发送给项目维护者。用户点击“生成反馈文件”后，应用在桌面写入 Markdown 文件；用户应先预览，再自行决定是否发送。

自动诊断可能包含稳定安装标识的前 8 位、应用版本、操作系统、语言、服务商状态、聚合使用计数、去标识的文件类型、性能统计、脱敏设置状态及脱敏日志。自动诊断不应包含案件名、当事人、文档正文、完整本地路径或原始文件名。用户主动填写在“用户描述”中的内容不会被自动删改，请勿粘贴不必要的个人信息。

## 6. 匿名使用遥测

遥测只有在编译时同时注入 `CASEBOARD_TELEMETRY_URL` 和 `CASEBOARD_TELEMETRY_KEY`，且不是测试构建时才会启用；任一配置缺失即静默禁用。公开 Windows 构建 workflow 声明了对应仓库 Secrets 的可选注入，但 Secrets 未配置、fork 或无权读取 Secrets 的构建不会启用遥测。

启用时，应用向配置的 Supabase REST 端点发送：

- 稳定设备 ID（本机生成的 UUID）；
- 每次启动随机生成的会话 ID；
- 应用版本；
- 粗粒度操作系统和架构，不含系统小版本；
- `session_start` 事件，以及约每 5 分钟一次的 `heartbeat`。

该遥测载荷不包含案件内容、当事人、文件名、API key 或用户填写的反馈。稳定设备 ID 仍可区分同一安装，不等同于“无法关联”的完全匿名数据。

## 7. 保留、删除与导出

- 本地数据通常会保留，直到用户在应用内删除、清理缓存，或手动删除应用数据目录；卸载应用不一定删除数据目录和用户导出文件。
- 部分删除可能采用软删除或保留关联记录，以维持审计、引用或恢复能力；对必须彻底清除的数据，请同时检查应用数据目录、原始案件目录、导出目录、备份和第三方服务端。
- 云端服务、元典、快递、飞书、滴答、Supabase 或自定义服务的保留与删除由对应服务商规则控制，CaseBoard 不能代替用户删除第三方已接收的数据。
- 导出只在用户主动操作后产生文件。导出物可能包含案件和个人信息，应由用户自行选择安全位置、访问权限和传输方式。

## 8. 用户责任与变更

律师案件资料可能包含高度敏感的个人信息、商业秘密和司法材料。用户应遵守适用法律、律师执业规范、客户授权和所在机构的安全制度，并在启用外部服务前完成必要评估。

数据处理行为随功能和代码更新可能变化。若实现与本文不一致，应以实际代码行为为准并及时修正文档。安全漏洞请通过仓库的 GitHub 私密漏洞报告渠道提交，不要在公开 Issue 中披露敏感材料。
