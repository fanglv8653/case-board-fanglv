## 四项范围

1. 日历只在 `HomeView mode="workspace"` 显示；不删除数据、不影响关键日期聚合。
2. 合同审查批注与修订作者使用律师姓名；时间使用本机当前时间，默认北京时间，OOXML 写入带 `+08:00` 偏移的 RFC3339 时间；同一次导出的全部批注与修订使用同一时间快照。
3. 飞书待绑定案件支持推荐、人工选择、确认绑定、解除、忽略和恢复；本地日期前缀只在匹配层归一化；飞书零写入，不改案件业务字段和文件夹名。
4. 非诉保持合同审查、合同起草、律师函三张主卡；合同审查吸收最小交易背景、附件完整性、法源/事实/律师复核状态和正式导出门禁；律师函只生成工作稿/正式稿，不自动发送，不混入民事和刑事文书。

## 公共契约

- 作者优先级：本次临时作者 > 已保存合同批注作者 > `user_display_name` > `合同审查（CaseBoard）`。
- 日期优先级：不允许前端传任意历史时间；后端导出开始时读取本机时间并转成带偏移 RFC3339，默认 Windows 北京时区应为 `+08:00`。
- 飞书绑定状态：`pending_binding -> bound -> pending_binding`；`pending_binding <-> ignored`；所有动作写本地审计。
- 自动匹配：唯一精确案号可自动；归一化名称必须同时通过领域及当事人/案由或罪名校验，歧义永远人工确认。
- AI/检索结果初始均为待律师复核；未满足门禁只能导出带待审核标识的工作稿。
- Skill 只转为结构化字段与门禁，不复制 Skill 或第三方提示词原文。

## 禁止事项

- 不写飞书，不申请额外写权限。
- 不重命名 Windows 案件文件夹，不覆盖原材料。
- 不把民事起诉状或刑事法律意见书放入非诉首页。
- 不修改旧工作树和历史 release 目录。
- Agent 只可提交 `submitted_for_review`，不得自判 accepted。

- 飞书绑定、解除、忽略、恢复可回退；正式业务表及非目标同步链接哈希不变；网络层无写请求。
- 工作台显示日历；民事、刑事、执行和案件详情不显示整块日历。
