# 安全政策

## 报告安全漏洞

**请不要通过 GitHub Issues 公开报告安全漏洞。**

如果你发现 CaseBoard 中的安全漏洞(尤其是涉及用户案件数据、本地文件访问、MCP 接入授权等敏感面的问题),请通过以下渠道私下报告:

- 🔒 通过 GitHub 私密漏洞报告(推荐):本仓库 **Security → Report a vulnerability**
- 💬 或加作者微信私下报告(二维码见 [README「License · 授权」节](./README.md#license--授权),注明 `[CaseBoard Security]`)

请在报告中尽可能提供:

- 漏洞描述与潜在影响
- 复现步骤(操作系统版本、CaseBoard 版本、操作序列)
- 你认为可能的修复方向(可选)
- 是否希望在公告中被致谢

我们会在 **5 个工作日内**响应初步确认,并在合理时间内给出修复时间表。

## 支持的版本

V0.1 之前的开发版本均不提供安全维护。
正式发布的版本会维护最近一个 minor 版本的安全更新。

| 版本 | 安全更新 |
|:---|:---|
| < 0.3 | ❌ |
| 0.3.x(当前) | ✅ |

## 数据隐私边界

CaseBoard 的设计原则:

- **本地存储**: 抽取后的结构化数据、报告、数据库均存在用户本机 SQLite / 文件
- **原文件只读**: 永不修改用户原始案件文件夹
- **云端处理须知**: 默认的文本抽取 / OCR 会把文档内容发往 DeepSeek / MinerU 处理(可改用本机模型);这是功能所需,非隐私缺陷
- **反馈通道脱敏**: 反馈仅生成本地 MD(版本/OS/堆栈/计数),案件标题/当事人内容**永不写入**,且由用户手动发送
- **MCP 接入授权**: 任何外部 AI 客户端必须在 App 内显式授权才能读写数据库

如果你发现任何违反以上原则的实现,请按上面流程报告。
