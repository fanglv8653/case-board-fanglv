# 贡献者公约

## 我们的承诺

为了营造一个开放和欢迎的环境,我们作为贡献者和维护者承诺,无论年龄、体型、身体健全与否、民族、性别认同和表达、经验水平、国籍、个人外貌、种族、宗教或性别取向,参与到我们项目和社区中的每一个人都不受骚扰。

## 我们的标准

有助于创造积极环境的行为包括但不限于:

- 使用友好和包容性语言
- 尊重不同的观点和经历
- 耐心地接受建设性批评
- 关注对社区最有利的事情
- 友善对待社区的其他成员

不能被接受的行为包括但不限于:

- 使用与性有关的言语或图像,以及不受欢迎的性骚扰
- 嘲讽、侮辱/贬损的评论,人身攻击或政治攻击
- 公开或私下骚扰
- 未经许可地发布他人的私人信息,比如住址或电子邮箱地址
- 其他在专业场合不适当的行为

## 我们的责任

项目维护者有责任为「可接受行为」的标准做出诠释,并采取适当、公正的纠正措施以应对任何不可接受行为。

项目维护者有权利及责任移除、编辑、拒绝违反本行为准则的评论、commits、代码、wiki 编辑、issues 和其他贡献,并暂时或永久性禁止任何贡献者表现出他们认为不适当、威胁、冒犯或有害的行为。

## 适用范围

当一个人代表项目或其社区时,本行为准则适用于项目所属的空间,也适用于公共空间。代表项目或社区的情形包括使用项目的官方电子邮件地址、通过官方社交媒体账户发布消息或在线上或线下事件中担任指定代表。项目维护者可以进一步定义和阐明项目的代表。

## 强制执行

可以通过项目维护人**微信**(二维码见 [README「License · 授权」节](./README.md#license--授权))或 GitHub 私密渠道联系,报告滥用、骚扰或其他不可接受的行为。所有的投诉都将被审查和调查,并将根据情况作出回应。维护团队有义务对事件的报告者保密。具体执行政策的更多细节可能单独发布。

不善意遵守或执行本行为准则的项目维护人可能会受到由项目领导其他成员决定的临时或永久的处分。
