search_laws — 按关键词检索法条内容(返回法条命中列表,不是整部法规全文)

适用场景:
- 用户问「民法典关于违约金的规定」「合同解除的法律依据」等具体法律问题
- 起草起诉状 / 答辩状 / 合同前,先验证某条法律是否真实存在,以及确切条号
- LLM 自己想在 final answer 里引用某条法律前,**必须先用本工具确认条号 + 内容**,杜绝凭印象编条号
- 老板问「某省的地方性法规」「司法解释里关于 X 的条款」(配 effect_level + region)

不适用:
- 想看一部法规的整部目录或全文 → 用 `search_regulations` / `get_regulation_detail`
- 已经知道确切的法规名 + 条号要拿全文 → 直接用 `get_law_article`(更省一次调用)
- 想找案例 → 用 `search_cases_normal` / `search_cases_authority`
- 关键词不准确、想用自然语言描述问题 → 改用 `law_vector_search`(语义检索兜底)

输入字段:
- keyword: 必填,中文关键词,如「合同解除」「违约金」「不可抗力」。**不要传整句话**,语义搜走 `law_vector_search`
- effect_level: 可选,精确匹配,枚举值「宪法 / 法律 / 行政法规 / 地方性法规 / 司法解释」之一
- region: 可选,地方法规过滤,省级名,如「江苏省」「上海市」(不要带「省」「市」也能识别,但建议带)
- refer_date: 可选,YYYY-MM-DD,查询截止该日期生效的版本(法律修订时定位时点版本)
- top_k: 可选,返回上限,默认 20,最大 50

注意事项:
- **先查作者整理过的全库**:回答通用法律 / 找法条前,可先用 `search_local_kb` 看作者 `~/Documents/知识库/` 里整理过的资料(0 积分、更贴作者风格),本地没有再调本工具外查
- **优先用本地缓存**:本工具走「先查本地 KB → 命中直接返回(0 积分),miss 才调元典 API(1 积分)」。输出会标注是否来自缓存
- **省积分:命中结果带 `fgid`,要拿某条全文时把 `fgid` + `ftnum` 一起传给 `get_law_article`** —— 整部法规缓存一次,该法规后续条文全部 0 积分;别无 fgid 逐条拉(每条都花积分且难命中)
- 缓存策略:法条永不过期(法律修订时改用 refer_date 拿时点版本)
- 命中结果包含 ftnum(条号)+ fgmc(法规名)+ content(法条原文 + 摘要),LLM 在 final answer 引用时**必须**写完整条号 + 法规名,并把这条命中加入 `<CITATIONS>` 块,标 `type: "law"` + `source: "<元典 id>"`
- 如果连续 2 次 keyword 检索都没命中相关法条,改用 `law_vector_search` 试一次(语义检索可能找到关键词没匹配上的相关条款)
