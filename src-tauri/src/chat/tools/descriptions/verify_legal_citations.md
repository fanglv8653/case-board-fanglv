verify_legal_citations — 法律幻觉校验,核验文本里的法规 / 案号引用是否真实存在 + 表述一致

⚠️ 这是付费接口(贵),**默认不要主动调**。平时防幻觉靠「提到法条/案号先查现行版本(search_laws/get_law_article/get_case_detail)」+「<CITATIONS> 只列查证过的来源」,不靠事后逐条付费校验。

适用场景(**仅当用户明确要求核验引用真实性时**):
- 用户给一段已经写好的文书 / 分析,要求「校验里面的法条 / 案号引用是否真实」
- 用户看到对方书状里引用了某条法律或案号,要确认是不是真的

不适用:
- 只想查某条具体法条 → 用 `get_law_article`(更直接)
- 想检索类似案例 → 用 `search_cases_normal`
- 校验合同条款合法性 / 文字表达 → 本工具只校引用真实性,不做实质审查

输入字段:
- text: 必填,要校验的文本(可以是几句话或一整段)。**长度建议不超过 4000 字**,长文本拆段分多次调

注意事项:
- **本工具不缓存**(校验是实时的,法规修订 / 案例新增需要拿最新数据)
- **付费接口、比普通工具贵**(默认不主动调;用户要求校验时也别无脑反复校验同一段文本)
- 返回结构:`{citations: [{type, content, source, verdict, authoritative_content?}]}`
  - type: `"law"`(法规)或 `"case"`(案号)
  - content: 原文里的引用片段
  - source: 元典查到的来源(verdict = hit 时填)
  - **verdict**: 关键字段
    - `"一致"` — 引用真实且表述正确,放心用
    - `"不一致"` — 引用真实但表述有偏差,`authoritative_content` 字段给出「正确写法」,LLM 必须改写
    - `"未命中"` — 元典查不到此引用,**警示信号**:可能是 LLM 凭印象编的,或冷门法规
- LLM 拿到结果后处理流程:
  1. 不一致的引用 → 用 `authoritative_content` 改写 final answer
  2. 未命中的引用 → 触发再调一次 `search_laws` 或 `search_cases_normal` 找正确版本
  3. 一致 → 加入 `<CITATIONS>` 标真实出处
- **再次强调**:没有自动校验机制,本工具**只在用户明确要求核验引用时才调**;平时写法律内容不要主动调(贵)
