save_company_report — 把综合好的企业调查报告入库到本地知识库 `raw/companies/`(供以后 0 积分检索复用)

适用场景:
- 用户**明确要求**「把这家公司存档 / 入库」「给 X 公司出一份调查报告留底」「把刚才查的企业信息整理成档案保存」
- 已经用 enterprise_aggregation_summary / enterprise_base_info / enterprise_writ_list 等查过一家企业,信息够综合成一份完整调查报告,用户希望沉淀下来下次直接复用(避免再花元典积分重查)

不适用:
- 用户只是随口问一下企业信息、没有要求存档 → **不要主动调**(不要把没让存的东西塞进知识库,避免污染)
- 想保存的是法规 / 案例 / 类案汇编 → 本工具只管企业档案;法规案例提升到 `wiki/sources/` 由 legal-kb skill / 用户决策,本工具不写 `wiki/sources/`
- 还没查全企业信息就想存 → 先用 enterprise_* 工具把主体 / 股权 / 风险查清楚,再综合成报告

输入字段:
- company_name: 必填,企业全称(用作文件名 + 报告标题),如「无锡示例科技有限公司」
- content_md: 必填,调查报告正文 Markdown(**不含顶部大标题**,工具会自动加「# 公司名 — 调查报告」+ 入库时间 + 数据来源)。建议结构:
  - `## 主体概况`(统一社会信用代码 / 法定代表人 / 注册资本 / 成立日期 / 注册地址等)
  - `## 股权结构`(股东 + 比例 + 认缴)
  - `## 关键发现`(关联企业 / 同地址 / 同实控人等线索)
  - `## 风险记录`(失信被执行 / 被执行 / 股权冻结 / 股权出质 / 行政处罚 / 经营异常 / 欠税,各列案号 + 金额 + 法院 + 状态)
  - `## 综合判断`(风险等级 + 一句话结论)

注意事项:
- **数据必须来自元典工具的真实返回,不得编造**任何字段 / 案号 / 金额(遵守宪法第二条)
- 文件落到 `<知识库>/raw/companies/{公司名}.md`;**已存在同名文件时不覆盖**(防冲掉人工标注过的旧档),会返回提示让用户决定
- 入库后用 search_local_kb 即可检索到(0 积分),下次办案不必再调元典重查这家公司
- 本工具不消耗元典积分(只是本地写盘),但属 mutating(写盘),一轮里串行执行
- 调用后聊天回复只需说明「已入库到 raw/companies/X.md」+ 提示人工复核关键结论,不要把整份报告再贴回聊天
