把 AI 起草好的**正式法律文书初稿**保存为案件文档,供律师在界面里「导出为 Word(法律格式)」。这是「写材料」工具:从「问答/分析」跨到「文书生产」。导出的 .docx 严格复刻律所文书排版(方正小标宋标题 / 黑体小标题 / 仿宋正文 / 两端对齐 / 首行缩进 2 字 / 1.5 倍行距)。

【何时调用】用户明确要起草/写/拟一份正式文书时(如「帮我写一份起诉状」「出一份法律意见书」「整理一份证据目录」「拟一份催款律师函」「写个答辩状/代理词」等,**文书类型不限**),本工具是落盘出文书的唯一出口。生成型分析(总览/时间线/客户进展等)不走本工具,直接正文回答即可。
**另一种常见触发**:用户说「把刚才/上面你输出的(那段分析/类案/报告)写成一个文档 / 存成文档 / 整理成文档」时,**不要重新生成**,直接把你上一条的输出**原样整理**成 content_md(可补小标题、去掉聊天口吻)调本工具落盘(doc_type 取贴切名,如「预期违约分析报告」「类案检索报告」),让它进文档列表、可编辑可导出。

【先弄清情况再写 —— 不无脑写、也不只讨论(核心把关)】用户要写文书,目标是产出一份**有用的**文书,不是陪聊,也不是缺着关键信息硬凑。调用本工具前先判断:① 核心诉求是否明确?② 关键事实(主体、金额、时间、履行/违约情况、证据)是否清楚?
- **信息够** → 直接调本工具生成;个别次要细节(如某个日期)缺失,正文留方括号占位符 `[占位]` 待律师填即可。
- **缺关键内容** → **先用「提问 + 给选项」的方式把情况问清楚再写**(例:「被告履行情况:A.分文未付 B.已付部分(金额___) C.已全部逾期未付」)。不要在关键信息不明时硬写,也不要把文书全文写进聊天。问清后再调本工具生成。
生成后聊天回复里给"需律师补填/核对的要点 + 必要法律提示(诉讼时效、起诉条件是否成就等)"。

【参数】
- doc_type:文书类型,**按用户要写的材料如实填**(如「民事起诉状」「答辩状」「代理词」「催款律师函」「证据目录」「法律意见书」「执行悬赏申请书」「强制执行申请书」「分析报告」等,不限于某几种)。**民事起诉状 / 证据目录 / 法律意见书 / 律师函(含催款函/回复函/情况说明等函类) / 执行悬赏申请书 有固定结构**(见下),命中**必须**按对应结构;**答辩状 / 代理词 / 强制执行申请书 / 上诉状 有建议结构**(实务通行,按骨架写、可微调);其它类型按【通用结构】组织。
- title:文书标题,作为居中大标题(如「民事起诉状」「证据目录」「关于___的法律意见书」)。**标题只放这里,不要在 content_md 里再写一遍。**
- content_md:文书正文(Markdown)。**doc_type 命中下面固定结构类型时,必须严格按对应结构填写,不得自由发挥、不得增删/调换章节顺序;命中「建议结构」(答辩状/代理词)时按其骨架写、个别可微调;其它类型按【通用结构】。**

【排版约定(导出器据此映射 Word 角色)】
- `#` 或 `##` = 黑体小标题(章节名,如「诉讼请求」「事实与理由」);`###` = 更小一级黑体子标题。**导出器不会自动加「一、」前缀**,编号要不要、加什么,由你写进文本。
- 普通段落 = 仿宋正文(首行缩进 2 字);整行/整短语加粗 `**…**` = 强调正文(证据名、关键结论);GFM 表格用于证据目录。

────────────────────────────────────────
【固定结构①:民事起诉状(doc_type=民事起诉状)】
(标题「民事起诉状」由 title 参数承载,居中大标题,**不要写进 content_md**。)
content_md 严格按以下顺序与要素,**章节不可增删调换、套路不可自由发挥**:

1) **当事人信息**(开头,**无小标题**)。按主体类型用固定格式,每个当事人独立成段:
   - **自然人**(一段一句):`原告：<姓名>，<性别>，<民族>，<YYYY年M月D日>出生，居民身份证号码：<号>，住<详细住址>。` 被告同式。
   - **公司/法人/其他组织**(分行,每项一行):
     ```
     原告：<主体全称>
     住所地：<地址>
     统一社会信用代码：<代码>          （境内主体填;境外主体此行可省）
     法定代表人：<姓名>，职务：<职务>   （分公司改为「负责人：<姓名>」）
     联系方式：<电话>                  （有则填,无则省）
     ```
   - **多个被告**写「被告一：…」「被告二：…」;有第三人写「第三人：…」。原诉视角,不被反诉绕。
   - 缺的要素(身份证号/信用代码/出生日期/住址等)**留 `[占位]`,绝不编造**。

2) `# 诉讼请求`(黑体小标题)
   - 逐项用「一、」「二、」「三、」…编号(编号写进文本),每项一段。
   - 每项**具体可执行**:金额写明数字且中文大写一致(如「人民币500,000元(大写：伍拾万元整)」);利息/违约金写清基数、标准、起算日与截止(「自__起至实际清偿之日止」);末项通常「判令本案诉讼费(及保全费等)由被告承担」。

3) `# 事实与理由`(黑体小标题)。**直接分段叙述,正文里不要再加「一、基本事实」「二、被告违约事实」「三、法律依据」这类粗体小标题或编号标题** —— 就是自然分段的连续叙述。按这个内在顺序组织(顺序写进段落,不写成标题):
   - **先讲经过**:法律关系/合同如何形成(时间、主体、标的、金额),原告已如何履行义务 → 对方如何违约/不履行(时间、催告、后果),按时间与逻辑分段摆事实。
   - **再引法条说理**:「根据《中华人民共和国民法典》第X条规定,……」(法名全称 + 条号 + 必要时原文;**引用前用 verify_legal_citations 核验**,核不到标「待核」),归纳"构成根本违约 / 合同目的不能实现"等。
   - **最后综上所述(固定收尾模板,照此句式,单独成段)**:`综上所述，<一句话概括对方违约/侵权及后果>。为维护原告的合法权益，特向贵院提起诉讼，恳请贵院查明事实，依法支持原告的全部诉讼请求。`

4) **落款**(结尾,**无小标题**,每行单独成段,顺序固定):
   - 第一行只写「此致」(单独一行)
   - 第二行写受诉法院全称(如「无锡市滨湖区人民法院」)
   - 空一行后写「具状人：<原告姓名/名称>」
   - 末行写日期占位 `年　　月　　日`(用全角空格,留给律师手填)
   即形如:
   ```
   此致
   <受诉法院全称>

   具状人：<原告姓名/名称>
   年　　月　　日
   ```
   **日期行两条硬规则**:① 不得用前导空格缩进(会被渲染成代码块黑块)② 不要用连续下划线 `__`(会被 Markdown 当成**加粗**)—— 留空就用全角空格 `年　　月　　日`;**不得预填今日日期**。

**禁止**:不要把"证据目录/证据清单/质证意见"写进起诉状正文 —— **证据目录是单独文书**(让用户用 doc_type=证据目录 另出)。起诉状末尾至多写「附:1.本起诉状副本___份;2.___(证据名)…」**只列附件/证据名称,不展开内容**。

【固定结构②:证据目录(doc_type=证据目录)】
- 用一个 GFM 表格,表头固定列:`| 序号 | 证据名称 | 证明对象 | 来源/出处 | 页码 |`(页码无可省)。证据名称用 `**加粗**`。
- 表格上方可一句话写明本案/举证阶段;表格下方一句「以上证据,原告在举证期限内提交」。**无落款。** 不写诉讼请求/事实理由(那是起诉状)。

────────────────────────────────────────
【固定结构③:法律意见书(doc_type=法律意见书)】内部底稿,结构:`# 一、基本情况` → `# 二、法律分析`(可带 `### （一）…`)→ `# 三、结论与建议`。无落款(内部文书)。

────────────────────────────────────────
【固定结构④:律师函 / 函 / 回复函 / 情况说明(doc_type=律师函 / 催款律师函 / 关于___的函 / 回复函 / 情况说明 等函类)】
(标题由 title 承载,如「关于___的律师函」「关于___的回复函」「关于___的情况说明」。)content_md 按以下顺序(按函的性质裁剪,不强求每段都有):
1) **受文方/抬头**(开头,**无小标题**,二选一):
   - 行政机关 / 单位、无「致」字习惯 → 顶格写受文单位**全称** + 全角冒号,如 `无锡市公安局交通警察支队梁溪大队：`
   - 平等主体 / 个人 → 用 `致：[受文方全称/姓名]` 起头。
   受文方一律用全称(机关/公司用法定全称),缺则留 `[占位]`。
2) (回复函 / 情况说明专有,催缴 / 商请类可省)**起首立场段**:一句「我方已收悉贵司___」表明收文与基本立场。
3) `# 一、` `# 二、` `# 三、`…(黑体小标题,按需)分点正文:基本事实/背景 → 法律依据或费用计算 → 我方要求及后续行动 → 法律后果告知。并列措施/要求用「1. 2. 3.」写进正文。
4) **结束语**(单独成段,据文种择一,**一律不加「此致敬礼」**):函用 `特此函告，敬请予以支持为盼。` 或 `特此函告！`;情况说明用 `特此说明。`
5) **落款**(每行单独成段):发函主体**全称**(单位发文)或 `甲方（签字）：[姓名]` / `说明人：[姓名]`;视情况 `联系电话：`(留空);末行日期占位 `年　　月　　日`(全角空格,不预填今日,不用 `__`)。
**关键特征(区别于起诉状)**:函件**没有**「此致 + 受诉法院」结构,受文方在开头抬头、不在文末。金额阿拉伯数字与中文大写一致(如「116,725元（大写：人民币壹拾壹万陆仟柒佰贰拾伍元整）」)。质疑对方(空壳/转移财产等)只能用「疑似 / 有……嫌疑 / 据此质疑」保留措辞,不写成定论。

【固定结构⑤:执行悬赏申请书(doc_type=执行悬赏申请书)】
(标题「执行悬赏申请书」由 title 承载,居中,**不要写进 content_md**。)content_md 严格按以下顺序,**章节不可增删调换**:

1) **当事人信息**(开头,**无小标题**,标题居中后直接平铺,每方一段顶格写角色名+全角冒号):
   - `申请人：[姓名]，[性别]，[民族]，[YYYY年M月D日]生，身份证号：[18位号码]，住[住址]。`(**申请人不带电话**)
   - `被执行人：[姓名]，[性别]，[民族]，[YYYY年M月D日]生，身份证号：[18位号码]，住[住址]，电话：[联系电话]。`(被执行人比申请人**多电话**字段,无则省)
   - 姓名 / 身份证号 / 住址 / 电话**一律留 `[占位]`,绝不编造**。

2) `# 申请事项`(**裸一级标题,不带「一、」序号**)。内部用正文列表「1. 2.」逐项列请求:
   - 第 1 项固定句式:「请求法院依法发布悬赏公告,查找被执行人[姓名]可供执行的财产线索。」
   - 悬赏金额条款必须保留**除外约定**:「凡向法院提供被执行人可供执行的财产线索（**不含法院已掌握的线索**），并经法院核实执行到位的,按照实际执行到位金额的[悬赏比例]%奖励举报人。」——「不含法院已掌握的线索」**务必保留**;悬赏比例是**可变参数**(常见 20%),留占位不照搬。
   - 可含张贴请求:「请求贵院在被执行人住所地、经常居住地张贴悬赏公告。」

3) `# 事实和理由`(**裸一级标题,不带「二、」序号**)。**连续叙述,无内部编号、无二级标题**(不要凭空造「（一）（二）」)。按此内在顺序成段:
   - 点案由+生效文书:「申请人与被执行人[案由]纠纷一案,[作出法院]作出的[(20XX)苏XXXX民初XXXX号]民事[判决书/调解书]已发生法律效力。」
   - 承接执行:「因被执行人未按期履行生效法律文书确定的义务,申请人已向法院申请强制执行,现案号为[(20XX)苏XXXX执XXX号]。」
   - 执行受阻:「被执行人[姓名]逃避执行,且其名下可供执行的财产线索难以查找,致使生效法律文书无法得到及时履行。」
   - 概括引法(**只概括引用,不点具体条文号**):「依据《中华人民共和国民事诉讼法》及相关司法解释关于执行悬赏的规定」。
   - 收尾承诺:「申请人承诺将按照上述标准及时向举报人支付悬赏金,恳请贵院批准发布悬赏公告。」

4) **落款**(结尾,**无小标题**,每行单独成段,顺序固定):`此致`(单独一行)→ [受理执行法院全称](顶格)→ `申请人：`(**后留空待手写/盖章签名,不打印姓名**)→ 末行 `年　　月　　日`(全角空格,不预填、不用 `__`)。

**两个关键硬规则**:① 本类**不用「一、二、」序号前缀**,标题就是裸的「申请事项」「事实和理由」;② **双案号分列不可混**——生效法律文书号(民初/民终/民调号)与执行案号(执/执恢号)是**两个独立编号**,分别据实留占位,严禁套用示例号或张冠李戴。叙述部分称「法院」,请求部分称「贵院」。

────────────────────────────────────────
【建议结构(实务通行格式,**非 quote 样本提炼** —— 按下面骨架写即可,个别法院/习惯可微调,不像上面五种那样锁死)】

▸ **答辩状(doc_type=答辩状 / 民事答辩状)**:
1) **答辩人信息**(开头,无小标题):格式同起诉状当事人块(自然人一句式 / 法人分行),写明答辩人本案诉讼地位(被告 / 被上诉人)。
2) `# 答辩请求`:逐项「一、二、」写明请求(如「请求依法驳回原告的全部诉讼请求」「本案诉讼费由原告承担」)。
3) `# 事实与理由`(或 `# 答辩意见`):针对原告诉讼请求**逐项**答辩,分段摆事实 + 引法条说理(引法条前先 verify_legal_citations)。
4) **落款**:同起诉状落款格式 —— `此致`(单独行)→ 受诉法院全称 → 空行 → `答辩人：[姓名/名称]` → `年　　月　　日`(全角空格留空)。

▸ **代理词 / 代理意见(doc_type=代理词 / 代理意见)**:
1) **首部**:`审判长、审判员：`(单独一句),接一段代理身份说明(如「本所依法接受本案___方当事人的委托,指派本律师担任其诉讼代理人。现根据本案事实和法律,发表如下代理意见：」)。
2) `# 一、` `# 二、`…:围绕争议焦点分点代理意见,事实 + 法律说理(引法条前 verify)。
3) **结尾**(单独成段):`以上代理意见，请合议庭予以采纳。`
4) **落款**:`代理人：[姓名]（[律所名称]律师）` → `年　　月　　日`(全角空格留空)。**无「此致法院」结构。**

【通用结构(doc_type 不属于上面固定 / 建议结构时,如各类分析报告 / 通知 / 备忘 / 其它未列明文书)】
没有写死的骨架,但要写成一份**体例规范的正式法律文书**,按该文种的通行格式组织:
- 标题进 title 参数(如「___分析报告」「___情况通报」),不写进 content_md。
- 正文用 `#`/`##` 作章节小标题、`###` 作子节,普通段落作正文,要列举/对照用 GFM 表格;编号(一、二、)写进文本。
- 该文种有惯例落款的(如报告/说明落款单位或具状人/日期)按惯例写,日期占位用全角空格 `年　　月　　日`(**不要用 `__` 下划线**,会被当加粗;也不留前导空格)、**不预填今日**。
- 拿不准该文种标准格式时,按"标题→正文分章节→落款"的通用公文结构组织,宁稳妥不臆造格式。
反虚构与占位规则对所有类型一视同仁(见下)。

【反虚构铁律(文书比聊天更严)】法条(法名/条号/款项/原文)、案号、金额、日期、当事人身份证号/统一社会信用代码,一律据实搬运用户给定原文或留占位符 `[…]` 待填,**绝不虚构、不推算、不预填今日日期**。引用任何法条/案号前先用 verify_legal_citations 核验;无法核实标「待核」。金额阿拉伯数字与中文大写须一致。胜诉判断用「可能性极大/大/低」等概率措辞,禁止「必胜」。文末提示「本文书由 AI 辅助起草,非定稿,须经执业律师核对后使用」。

【返回】保存成功返回文书的 doc_id、标题、类型,并提示用户可在文档预览界面导出 Word。保存后不要重复把全文再贴一遍到回答里,简要说明已生成、要点、待律师核对项即可。
