semantic_search_case_docs — 按「语义/含义」检索本案材料全文,返回最相关的若干片段

跟 `find_in_document`(精确关键词)是互补关系:**不确定该搜哪个词、想按主题/意思找**时用本工具;**已知确切关键词、要定位精确位置**时用 `find_in_document`。

适用场景:
- 想找「跟某个争议点/事实有关」的材料,但不确定文件里用的是哪个词
  - 例:查「被告承认欠款」——文件里可能写成「确认尚欠」「认可未付」「同意支付」,关键词搜不全,语义检索能召回
- 用户问「关于交付时间有什么约定」「哪些材料涉及违约责任」「对方有没有自认」这类**主题型**问题
- 一上来不知道关键证据散落在哪几份文档里,先用语义检索快速定位候选,再用 `read_case_doc` 读全文细看
- 跨多份文档找「最相关的那几段」,而不是逐份 `find_in_document`

不适用:
- 已知确切关键词/人名/金额/案号,要精确定位 → 用 `find_in_document`(更快更准,不耗 embedding 额度)
- 想读某份文档完整内容 → `read_case_doc`
- 不知道有哪些文档 → 先 `list_case_docs`
- 搜法条 / 类案 / 本地知识库 → 分别用 `search_laws` / `case_vector_search` / `search_local_kb`(本工具只搜**本案**材料)

输入字段:
- query: 必填,用自然语言描述你想找的内容(写完整意思,别只写一个词;如「被告在聊天记录里承认欠款的陈述」)
- top_n: 可选,返回最相关的几个片段,默认 6,最大 12

返回结构:`[{doc_id, filename, category, score, excerpt}]`,按 score(余弦相似度,越大越相关)降序。
- 拿到片段后,如需完整上下文,用返回的 `doc_id` 调 `read_case_doc` 读全文。
- 写进回答的事实/引用,按宪法附录 A 用 `type:"doc"` 标注来源(source 写 filename)。

注意事项:
- **不消耗元典积分**(走本地向量索引 + embedding API;默认 bge-m3 免费额度)。
- 只检索「材料文档」:AI 生成的报告、律所程序/归档类材料(风险告知/谈话笔录等)不在索引内 —— 跟喂进上下文的材料一致。
- 索引懒加载 + 增量缓存:首次检索某案会稍慢(要把全文 embed),之后文件没变就秒回。
- 若本案未配置 embedding(用户没在设置填 key),本工具会提示「未配置」并建议改用 `find_in_document` —— 此时直接退回关键词工具即可,别反复调本工具。
