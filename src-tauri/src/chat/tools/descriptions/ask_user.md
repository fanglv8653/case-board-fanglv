向用户发起「选项式追问」。当你要起草正式文书(起诉状 / 意见书 / 函等)或执行某项工作,但**关键信息缺失、无法写出有用结果**时,调用本工具把问题抛回给用户,而不是凭空硬写、也不是在正文里堆一长串问句。

前端会把每个问题渲染成**可点击的选项卡片**(像 Claude Code 的提问框):用户点一下选项,或在自由输入框里填一句话,就把答案回灌给你,你下一轮再基于答案继续(写文书 / 接着分析)。这比让用户对着一段文字手敲回答顺手得多。

**什么时候用**:
- 起草文书前,核心要素不清楚 —— 原告是个人还是公司?被告履行情况(分文未付 / 已付部分 / 全部逾期)?是否已过约定付款期限?主张的具体金额 / 利息起算点?诉讼请求到底要哪几项?
- 用户指令本身有歧义,有两三种合理理解,选错方向会白做。
- 缺一个能决定整篇走向的关键事实。

**起草文书场景(重点):倾向多问一点,但给出口、别无限问**:
- 起草正式文书前,**倾向于先问 1-2 轮搜集背景**(主体细节、诉求范围与金额、关键事实与时间线、对方履行/抗辩、手上证据等)—— 问得越准,文书越有用,不要拿到最少信息就急着写。
- **每轮都给用户一个「直接写」的出口**:在某个问题的选项里加一项「以上信息够了,直接起草」,或单列一问「是否还要补充更多细节?」给选项「继续补充 / 够了,直接帮我写」。用户选这个出口就立即 save_artifact、不再问。
- **但绝不没完没了**:问过 1-2 轮、三类核心要素(① 身份能识别 ② 核心诉求清楚 ③ 关键事实清楚)都明确了,就直接 save_artifact,**不要非等用户点「够了」才停**。详见宪法第六条。

**什么时候不要用(别滥用)**:
- **次要细节别问,留 `[占位]` 让律师补**:受诉法院、利息/违约金计算口径与起算日、某个具体日期、当事人民族/籍贯等次要身份字段。
- **案件快照/材料里已经有的信息绝对不许再问**(例如快照已写民族「汉族」,就别问)—— 先看清楚已有什么。
- 答案能自己从案件材料里查到 —— 先用 read_case_doc / find_in_document 去查,别把能自查的事推给用户。
- 纯粹想确认「我理解对吗」这种客套 —— 不必问。

**怎么问**:
- 一轮 2-4 个关键问题,别一次堆十个轰炸用户;问完这轮拿到答案,若还缺再问下一轮(总共别超过 1-2 轮)。
- 每个问题尽量给好预设 `options`(2-4 个常见答案,如 `["个人","公司"]`、`["分文未付","已付部分","全部逾期未付"]`),让用户点一下就行;起草类记得带上「够了,直接写」这个出口选项。
- 选项无法穷尽、需要用户填具体内容(姓名 / 全称 / 金额 / 日期)时,把该问题的 `allow_input` 设为 `true`,用户会看到自由输入框。
- 调用本工具时,正文(content)里只写**一句**自然的引导语(如「为把这份起诉状写准确,我先和你确认几点」),**不要**再把问题清单也抄进正文 —— 问题与选项放进本工具参数即可。
