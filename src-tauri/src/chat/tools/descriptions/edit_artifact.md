对**已经生成过的文书**做局部修改:只替换其中一小段,而不是重写整篇。

何时用:用户对你之前用 `save_artifact` 生成的文书(或界面里正在编辑的那份)提出**局部改动**——「把第二段的金额改成 80 万」「这里的『2023 年 5 月』改成『2024 年 3 月』」「删掉最后那句」「把『被告应当』改成『被告依法应当』」「在诉讼请求里再加一条」等。这种情况**一律用本工具**,不要重新调 `save_artifact` 把整篇文书再吐一遍——重写既慢又贵,还会无意改动其它本不该动的内容。

参数:
- `doc_id`:要改的文书 document id。用刚才 `save_artifact` 返回结果里的那个 doc_id;若系统提示里标了「当前编辑文书」,用它的 doc_id。
- `find`:文书里**要被替换掉的那一段原文**,需与文书内容**逐字一致**(标点也要对)。可以从你之前生成的内容里原样复制。Markdown 的加粗标记 `**` 可省略——系统会自动做「忽略加粗/空白差异」的归一匹配。片段要写得**足够唯一**(够长、带上下文),否则文书里出现多处相同文本时无法定位。
- `replace`:替换成的新内容。**只写这一段的新文本**,不要把整篇重写进来。允许传空字符串,表示**删除** `find` 这一段。
- `context_before`(可选):`find` 紧邻的前一小段文字。当文书里有多处相同 `find` 时,用它指明改的是哪一处。
- `context_after`(可选):`find` 紧邻的后一小段文字,同样用于消歧。

失败处理:若返回「没找到要替换的片段」,说明 `find` 跟文书原文对不上——把它改成文书里逐字一致的一段(从你生成的内容里复制最准),或补 `context_before` / `context_after` 锚点后重试。若返回「出现多处」,说明 `find` 不唯一——把它写长一点或加 context 锚点。

成功后只会回给你改动处附近的一小段新内容供你确认;**不要把文书全文再贴回聊天**。同一份文书要连续改多处,就多次调用本工具(同一个 doc_id),不要攒成一次重写。
